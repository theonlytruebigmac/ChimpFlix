ChimpFlix — Admin redesign mockups

A unified pass over the admin surface, anchored on the four UX observations from the round-3 review: status sprawl, raw window.confirm, 1k-line monoliths, and per-form save UX drift. Brand tokens (#0d0d0d / #141414 / #e50914) carry over from the viewer side so the admin reads as part of the same product, not a separate console.

1. Home 2. Libraries 3. Users 4. Server settings 5. Scheduled tasks
How to read these — every mock uses the same chrome (top bar, sidebar, page head, content) and the same primitives (cards, pills, tables, drawer, save bar). Anything that would be a one-off component today is collapsed into a shared primitive on purpose, because the value of this redesign is in the system, not the individual screens.

Open questions for review:
  1. Does the unified Home collapse the right pages, or are Stats / Library Health pulling enough weight to deserve their own landings?
  2. Is the list + drawer pattern (Libraries & Users) preferred over the current full-page detail navigation, or does it hide too much behind a click?
  3. For settings forms: sticky save bar with explicit dirty count vs. inline per-section save buttons (today)?
  4. Should Simple be the Scheduled-Tasks default, or keep Advanced as the landing for operators who know what they want?
  5. Color palette: the accent-red is already in --color-accent; should we shift to a less Netflix-coded admin accent (e.g. cyan / amber) so the viewer / admin halves read as distinct contexts?

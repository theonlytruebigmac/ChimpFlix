Settings & Admin · v2 IA proposal

Everything after /settings, restructured.

The app keeps growing — 5 user tabs and 27 admin pages across a deep sidebar. It all works, but it doesn't flow. This set keeps the Netflix look you already have and rethinks the structure: how the areas are grouped, how you move between them, and how each value is presented. Click through — every mockup is a live, navigable prototype sharing one shell.

Open the prototype → Jump to the admin console ⌘K  command palette on every page
What's changing
01 · NAVIGATION

Two contexts, one shell

A clear switch between You (personal settings) and Server (the admin console). Same chrome, same patterns — no more "where did that live again?"

02 · DEPTH

27 admin pages → 11 sections

Related pages collapse into one section with in-page tabs (Tasks = Overview·Queue·Activity·Flow). The sidebar gets short and scannable.

03 · FINDABILITY

Search any value (⌘K)

A command palette jumps straight to a page, a single setting, or an action. "So much" stops being a problem when you can type to it.

04 · PRESENTATION

One row pattern everywhere

Label + helper on the left, the control on the right, hairline between. Forms get a sticky save bar; lists get a master-detail drawer.
